# AIDrivenCompany Vision

## What This Is

AIDrivenCompany is a Company OS. Founders use it to create, simulate, and operate entire companies end-to-end through AI. It is not a product builder. It is not a dashboard. It is not a collection of disconnected tools stitched together. It is the operating system for a company that does not exist yet -- or one that needs to evolve.

A founder can arrive with nothing -- no idea, no plan, no team -- and leave with a running company. Or they can arrive with an existing company and use the system to simulate their next move before committing to it.

AIDrivenCompany is open source under the MIT license, with a hosted version available from day one.

## North Star

**How will companies be created in the future?**

Through AI-native systems where ideas are simulated, executed, and evolved continuously within a unified Company OS. Not by assembling tools. Not by hiring before validating. Not by guessing.

## The Core Loop

Every company inside AIDrivenCompany follows one loop:

**Simulate -> Build -> Launch -> Learn -> Adapt -> Repeat**

Nothing ships without simulation. Nothing stays without proof. The system enforces this loop at every level -- from a single campaign to an entire go-to-market strategy.

## Three Layers

The product is built on three layers, each with a distinct responsibility. They are not optional. They are not interchangeable. They are hierarchical.

### 1. Strategy Layer (top)

This is what makes AIDrivenCompany a Company OS rather than another automation tool.

- **Genesis**: Turn a blank slate into a company. Idea generation, validation, ICP discovery, positioning, naming -- all connected.
- **Simulation**: Every decision is modeled for downstream impact before it is committed. Change your pricing? See the effect on margins, messaging, and campaign copy before it happens.
- **Company Graph**: A living, connected graph of everything -- idea, ICP, features, pricing, channels, campaigns, proof, metrics. Nothing exists in isolation.
- **CEO Dashboard**: A single surface where the founder sees the state of the company, pending decisions, and the consequences of each option.

### 2. Orchestration Layer (middle)

Manages the AI workforce that runs the company.

- Agent org charts with defined roles and reporting lines
- Budgets and resource allocation per agent
- Goals, governance, and guardrails
- Heartbeats and health checks across all running agents
- Task delegation and escalation

### 3. Execution Layer (bottom)

AI agents that do the actual work across real channels.

- Send emails, post to social media, respond to messages
- Build landing pages, write copy, generate assets
- Run campaigns, track conversions, collect feedback
- Execute product workflows end-to-end

## Three System Layers

Across the entire product, work flows through three system layers:

| Layer | Scope |
|---|---|
| **Strategy** | Decisions, simulation, vision, dependencies |
| **Execution** | Product, GTM, campaigns, workflows |
| **Learning** | Proof, metrics, feedback, adaptation |

Strategy defines what to do. Execution does it. Learning determines whether it worked and feeds back into Strategy. The loop is closed. Always.

## Key Differentiators

**Simulation-first.** Other tools let you do things. AIDrivenCompany lets you see what happens if you do things -- before you do them. Every decision flows through the Company Graph, so changing one node reveals the impact on every connected node.

**Starts from zero.** Most tools assume you already know what you are building. AIDrivenCompany supports founders who have nothing but the desire to start something. Genesis takes them from blank page to operating company.

**Company Graph.** Ideas, customers, features, pricing, channels, campaigns, proof, and metrics are not separate databases. They are one connected graph. This is what makes simulation possible and what makes the system coherent.

**Company OS, not a tool.** The UX is designed so that founders feel like they are operating a real company -- not using software. The system is the company's operating layer, not an accessory to it.

**Multi-company workspaces.** One founder, many companies. Each with its own graph, its own agents, its own loop. Switch between them like switching tabs.

## Open Source + Hosted

The full product is open source under MIT. Anyone can run it, fork it, extend it.

The hosted version removes infrastructure burden and adds managed services. Both versions share the same codebase. There is no open-core bait-and-switch -- the open source version is the real product.

## Guardrails

These are non-negotiable. If we drift into any of these, we have lost the plot.

1. **Never become a product builder.** We are not Vercel, not Replit, not a website generator. We operate companies. Building artifacts is what the Execution Layer does in service of the Strategy Layer.

2. **Never break the graph.** Every feature must connect to the Company Graph. If something cannot be connected, it does not belong in the product.

3. **Never skip simulation.** No decision path should allow a founder to commit to a change without seeing its downstream impact. Simulation is not a feature -- it is the foundation.

4. **Never assume the founder knows what to build.** The system must be equally useful to someone with a fully formed business plan and someone with nothing but a vague ambition.

5. **Never ship disconnected tools.** Every capability must be part of the loop. If it does not feed back into Simulate -> Build -> Launch -> Learn -> Adapt -> Repeat, it is not part of this product.

6. **Never gate the open source version.** The hosted version can offer convenience, not capabilities that the self-hosted version lacks.
